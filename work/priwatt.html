CASE STUDY · 2024–PRESENT

priWatt — Design System
from Brand Book to Production

priWatt is bringing solar energy to homeowners across Germany, and their growing product suite needed a solid foundation to scale. With only an agency-delivered brand book to start from, no design system existed — slowing delivery and creating inconsistency across touchpoints. We built a production-ready design system from the ground up: a three-tier token architecture, typography scale, responsive grid, and component library that cut design-to-dev delivery time by 75%.

01 · Context

priWatt's brand was delivered by an external agency as Figma files — colour palettes, typography, and visual direction. No tokens, no component specs, no grid. Just a brand.

I took that foundation and built the entire design system from scratch. Once it was in place, the development team had everything they needed to build from — a single source of truth from day one.

02 · The Problem

Working without a design system created four compounding problems that slowed down both design quality and delivery speed.

PROBLEM 01
Brand book ≠ design system
The agency delivered colour palettes and typography in Figma — not tokens, not component specs, not anything a developer could implement directly. Everything between brand guidelines and production had to be figured out manually, every time.
PROBLEM 02
No shared design language
Without a system, every designer interpreted the brand independently. Buttons varied. Spacing was guesswork. Colour names like "Electric Lime" carried no semantic meaning — they described a swatch, not a role.
PROBLEM 03
Handoff had no structure
With no tokens or component documentation, every dev handoff turned into a back-and-forth. The same questions came up repeatedly — padding values, interactive states, breakpoint behaviour. Time lost every sprint.
PROBLEM 04
Nothing could scale
Every new page or campaign started from zero. No reuse, no consistency enforcement. A single brand change meant touching every file by hand — not a process that could grow with the product.
03 · Outcomes

Establishing the design system created measurable improvements across speed, consistency, and design-dev collaboration.

04 · Process

Building from a brand book required a deliberate sequence — from audit to architecture to implementation.

01
Brand audit
Mapped every colour, typeface, and spacing value from the brand book. Identified what was missing: semantic roles, interactive states, responsive behaviour. Built a gap analysis against production needs.
02
Token architecture
Designed a three-tier token system: Primitive → Semantic → Component. Raw values live only at the primitive level. Everything above references tokens — never hardcoded values.
03
Token Studio implementation
Built the full token set in Token Studio — colour, typography, spacing, border-radius, shadow, motion, z-index. Exported as a structured JSON file, version-controlled and consumable by dev.
04
Component library
Built the Figma component library on top of the token system. Every component references semantic tokens — not primitives. Variants, states, and responsive behaviour all documented.
05
Handoff & adoption
Delivered structured JSON to dev, Figma library to designers. Ran onboarding sessions. Wrote usage documentation. Measured adoption through reduction in handoff questions and revision requests.
05 · Token Architecture

The core design decision: a strict three-tier hierarchy where values flow in one direction only — primitives define raw values, everything else references tokens. This makes global changes a single-point update.

Tier 01
Primitive
Raw values. Named after what they are, not what they do. Never used directly in components.
color.lime.100 color.forest.800 spacing.4 → 16px radius.full
Tier 02
Semantic
Context-aware names. Describes role in the UI — not the visual value. Theme-swappable.
color.bg.default color.text.primary color.border.subtle color.action.primary
Tier 03
Component
Scoped to a specific component. References semantic tokens — never primitives directly.
btn.bg.default btn.text.default input.border.focus card.bg.elevated

07 · Typography Scale

The brand book specified Roobert as the primary typeface across all weights — Bold for headlines, Semibold for subheads, Regular for body. I formalised this into a systematic scale with defined size, line-height, and letter-spacing tokens at each step.

type.display 48px / −2% Display Heading
type.h1 40px / −1% Heading One
type.h2 32px / −1% Heading Two
type.h3 24px / 0% Heading Three
type.body-lg 16px / +1% Body large — product descriptions, lead copy
type.body 14px / +1% Body default — UI labels, interface text
type.caption 12px / +1% Caption — helper text, metadata, tags
type.data 11px mono data.label → token.name → 16px

09 · Token Delivery

The full token system was exported from Token Studio as a structured JSON file — version-controlled, consumable directly by engineering. One source of truth for both Figma and production.

tokens.json · Token Studio export 
// base › spacing
"spacing": {
  "4": { "$value": "16px", "$type": "spacing", "$description": "16px — base unit" },
  "8": { "$value": "32px", "$type": "spacing", "$description": "32px" },
  "32": { "$value": "128px", "$type": "spacing", "$description": "128px — section gap" }
},
// base › border-radius
"border-radius": {
  "none":  { "$value": "0px" },
  "sm":    { "$value": "4px", "$description": "buttons, inputs — primary UI" },
  "md":    { "$value": "8px", "$description": "cards, modals, chips" },
  "lg":    { "$value": "12px", "$description": "large surface areas" },
  "full":  { "$value": "9999px", "$description": "pill — tab chips only" }
},
// base › typography
"font-size": {
  "md":  { "$value": "14px", "$description": "Medium — body default" },
  "lg":  { "$value": "16px", "$description": "Large — body large" },
  "5xl": { "$value": "32px", "$description": "5XL — heading large" }

10 · Website & Product
01 / 02
Advanced PV System Configurator
The core UX challenge: solar system selection requires technical knowledge most consumers don't have. I designed a real-time configurator that integrates geospatial data and electrical logic — letting users design and validate their own system in minutes. The tool hides complexity behind a guided, consumer-grade interface while producing technically accurate outputs.
02 / 02
Framer Marketing Hub
Marketing campaigns — Black Friday, Christmas, new product launches — previously required full custom development per campaign. I built a centralised Framer Marketing Hub: a DS-backed suite of plug-and-play sections that the team can drag, drop, and deploy without engineering involvement.
